Stony Creek Water Quality and Hudson Field Station Meteorological Data

Data Dictionary

Author

Ilianna Ditren, Natalie Novoseltseva, Joy Terentiev

Data Source

In this data exploratory analysis, we analyzed water quality data from the Stony Creek tributary in the Hudson River and meteorological data from the Hudson River Field Station to study water quality and conditions in relation to weather.

To perform the analysis, we used Stony Creek water quality data for the years 2002-2021 and Hudson Field Station weather indicator data for the years 2001-2026. Raw water quality and weather data was obtained from the Centralized Data Management Office (CDMO) website (https://cdmo.baruch.sc.edu).

Below are the detailed steps to obtain the data:

1.  Navigate to the CDMO website.

2.  Click on the Get Data tab at the top of the page.

3.  Select Advanced Query System and click Launch.

4.  Locate the Zip Downloads option and click Choose ZIP Files.

5.  From the list of National Estuarine Research Reserves, select Hudson River, NY.

6.  Select the appropriate dataset:

- For weather data from the field station, select hudfsmet-p.

- For Stony Creek water quality data, select hudscwq-p.

- For Tivoli North water quality data, select hudtnwq-p.

File Access and Location

All raw data files containing data for all years are located in the data folder of the R-Studio Project. Inside the data folder, you can also find folders containing a csv file per year of observation of water quality data on Stony Creek and Tivoli North, and meteorological data from Field Station.

In that folder, you can also find the README file containing the information present in this document.

Quality Control

Each variable in both datasets contains a flagged column. The flag column, F_, contains a quality control (QC) flag and any additional QC codes that apply to the corresponding parameter data. Refer to the lists below for the available QC flags and codes and their descriptions.

Quality Control Flags

Column Description
-5 Outside high sensor range
-4 Outside low sensor range
-3 Data rejected due to QA/QC
-2 Missing data
-1 Optional parameter not collected
0 Passed initial QAQC checks
1 Suspect data
2 Reserved for future use*
3 Calculated data: non-vented depth/level sensor correction for changes in barometric pressure*
4 Historical: Pre-auto QA/QC
5 Corrected data

*2 and 3 Flags were used in 2007 and 2008 to designate data that were outside 2 and 3 standard deviations from the historical seasonal mean. This automated check has since been discontinued and the 2 and 3 flags applied to the data during this time are being removed from the dataset as the data are authenticated.
